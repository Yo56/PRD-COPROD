8 // Normally we don't export class templates (but do complete specializations),
9 // inline functions, and classes with only inline member functions. Exporting
10 // classes that inherit from non-exported/imported bases (e.g., std::string)
11 // will end up badly. The only known workarounds are to not inherit or to not
12 // export. Also, MinGW GCC doesn't like seeing non-exported function being
13 // used before their inline definition. The workaround is to reorder code. In
14 // the end it's all trial and error.
